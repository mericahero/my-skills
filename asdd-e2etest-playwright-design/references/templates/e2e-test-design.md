## 截图与视觉证据策略

默认不设计截图断言。截图只作为人工诊断或视觉类场景的辅助证据，不能替代 URL、文本、role、表单状态、网络响应和控制台错误等结构化断言。

| 场景 | 是否允许截图 | 用途 | 存放位置 |
|---|---|---|---|
| 常规业务流程 | `否` | `使用结构化断言证明业务结果` | `无` |
| 失败现场概览 | `可选` | `辅助人工判断白屏、错误页面、遮罩、弹窗状态` | `playwrightE2E/artifacts/{run_id}/screenshots/` |
| 视觉契约 / 布局回归 | `可选` | `证明布局、响应式、主题、关键视觉状态` | `playwrightE2E/artifacts/{run_id}/screenshots/` |
| Canvas / 图表 / 地图 / PDF 预览 | `可选` | `补充 DOM 难以表达的视觉输出` | `playwrightE2E/artifacts/{run_id}/screenshots/` |

## 数据隔离策略

| 策略 | 说明 |
|---|---|
| 唯一数据 | 所有创建类操作使用 `timestamp + random` 生成唯一数据（用户名、邮箱、手机号） |
| API 清理 | `afterAll` 中通过 API 删除创建的测试数据 |
| 禁止硬编码 | 禁止硬编码邮箱、手机号等有唯一约束的字段 |
| 数据生成函数 | 使用 `ts()` 函数生成唯一标识：`Date.now().toString(36) + Math.random().toString(36).slice(2, 6)` |
